Privacy

Privacy-first direction

Textezi is intended to keep its core text-conversion flow on-device and avoid sending user text to the developer by default.

Core principles
User text should remain on the device for the core workflow.
No custom keyboard is required for the product direction.
Saved preferences and overrides should remain local in the MVP direction.
Analytics, if added later, should avoid raw user text.
