Privacy

Local-first by product design

The core Textezi converter is designed to run on-device. The app’s main romanization path, selected-language rules, and saved corrections are built to stay local to the phone.

Core conversion
Textezi’s main script-conversion flow is on-device.
The app keeps the language the same and rewrites only supported script text into Roman output.
Enabled languages, priority order, and transliteration behavior for the local core path stay on the device.
Accessibility boundaries
Accessibility automation is optional and requires explicit Android permission plus an in-app disclosure step.
Textezi is designed to act only on committed text in editable fields you are actively using.
Passwords, OTPs, phone fields, account-sensitive inputs, and similar protected forms are intentionally excluded.
Local saved data
Selected languages, automation preferences, floating bubble settings, and disclosure acceptance are stored locally on the device.
Manual corrections and learned preferred spellings stay on the device as part of the app’s quality loop.
The current access model also stores local device state for the soft 30-day trial and permanent unlock status.
External services
Google Play billing is used for the one-time permanent unlock where billing is configured and available.
The public website and GitHub support links are external services outside the core converter itself.
The app may include internet-related integrations for product services, but the core converter does not require a backend round-trip to work.
