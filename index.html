What Matters

Product decisions that make Textezi feel reliable

Textezi is intentionally narrow. It focuses on script conversion, keeps the workflow understandable, and avoids touching text that should stay as-is.

Selective by default

Textezi only converts languages the user has turned on. That keeps the output intentional instead of overreaching.

Same language, different script

The goal is Roman readability, not rewritten meaning. Textezi changes presentation, not the language itself.

Fits real workflows

Manual conversion, suggestion flows, and automation options can coexist so users can choose how involved they want the app to be.

Local-first direction

The core experience is designed to stay on-device, with preferences and learned behavior kept close to the user.

Scope

A careful launch surface with room to expand

The current release is intentionally controlled. Textezi is built to grow language by language, but the public story should stay disciplined and credible.

Today

Current release posture

Launch support begins with a smaller verified set inside the app.
Public messaging stays product-level unless coverage needs to be named directly.
Reliability matters more than claiming broad language support too early.
Direction

System direction

Add support deliberately, one language system at a time.
Keep user control at the center of what converts and what stays untouched.
Preserve a clear line between romanization, translation, and keyboard replacement.
